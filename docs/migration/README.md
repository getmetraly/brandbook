# Migration Backlog Index

This folder contains the execution-ready backlog derived from the brandbook migration-readiness audit and the component migration plan.

## How To Use This Index

1. Start with the baseline backlog to freeze the current state.
2. Move to the foundation and interaction backlogs to stabilize the reusable base.
3. Use the surface backlog for cards, tables, badges, empty states and widget chrome.
4. Use the dashboard backlog for the highest-risk product flow.
5. Use the website backlog when aligning public pages and marketing primitives.
6. Use the docs backlog to keep the portal canonical and navigable.
